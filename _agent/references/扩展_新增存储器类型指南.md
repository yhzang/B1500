# 如何给上位机新增一种存储器类型(RRAM / PCM / MRAM …)

> 版本 v1.0 / 2026-06-23 / claude
> 配套:`B1500_GUI架构设计_PySide6.md` §2(共性壳 vs 适配层)、M2 计划 §2。
> 一句话:**`gui/` 壳一行不改;只在 `src/fefetlab/` 加协议 + 在 `gui/adapters/` 加画法。**

---

## 0. 为什么不用改壳

整套上位机切成两层:

- **共性壳 `gui/`**(主窗口、协议树、参数表单、运行控制、绘图面板、日志、历史浏览、
  线程层)——只认抽象 `REGISTRY / ParamSpec / csv_schema / EngineCallbacks`,
  **不写任何具体存储器/协议码**。协议树按 `ProtocolSpec.family` 泛化分组,
  参数表单按 `ParamSpec` 自动生成,绘图按 `csv_schema` 查 `plot_dispatch` 注册表分派。
- **适配层**——每种存储器一份:runner(打波形/测量)+ `ProtocolSpec` 注册 + 绘图器。

`tests/test_extensibility_seam.py` 用一个假 family(RRAM)证明:注册一个新 family 的
`ProtocolSpec` + 一个 `@register_plot(schema)`,协议树自动多出一组、新 schema 自动能出图,
**壳层无需改动**。下面就是把这条缝走通的实操清单。

---

## 1. 五步加一种存储器

### ① 写 runner(唯一必写的"物理代码")
在 `src/fefetlab/protocols/` 下新建模块(如 `rram.py`),实现一个或多个
标准签名的 runner:

```python
def run_stage_rram_forming(backend, view, *, callbacks=None) -> StageSummary:
    # 用现有积木:_build_read_phase / _configure_and_run_phase / DCMeasurePoint …
    # 落盘统一走 orchestration.export.write_rows_csv + write_manifest_yaml(UTF-8 无 BOM)
    ...
```
- 复用现有积木,**不要新造执行器**;落盘列名沿用 `FIELDNAMES`(或为新 schema 定义自己的列)。
- 安全:沿用 `StopGatePolicy`(停门度量列按你的数据选,如 RRAM 看 compliance / I_reset)。

### ② 注册 `ProtocolSpec`(声明协议有哪些、参数是什么)
在 `src/fefetlab/engine/registry.py` 的注册流程里,把新协议升格成 `ProtocolSpec`:

```python
ProtocolSpec(
    id="RRAM_FORMING", title="Forming 成形", family="RRAM",     # ← 新 family
    physics="forming", description="…",
    params=(ParamSpec("v_form", "成形电压", ParamKind.FLOAT, 3.0, unit="V"), …),
    csv_schema="rram_iv",                                       # ← 新 schema
    runner=run_stage_rram_forming,
)
```
- `family="RRAM"` 让协议树自动多出 **RRAM 分组**。
- `params` 让参数表单自动生成控件(kind→控件、unit→SI 缩放、visibility→三档显隐)。
- `csv_schema="rram_iv"` 决定结果用哪个绘图器。

### ③ 写绘图器(把该 schema 的 CSV 画成有意义的图)
在 `gui/adapters/` 下新建 `rram_plots.py`:

```python
from gui.plot_dispatch import register_plot

@register_plot("rram_iv")
def plot_rram_iv(df, plot_widget, *, live: bool, options=None):
    # 画 R vs cycle / I-V 回滞 …
    ...
```
然后在 `gui/adapters/__init__.py` 里 `from . import rram_plots` 触发注册。
**这是唯一会碰 `gui/` 的地方,且只在 `adapters/` 子目录(适配层),不是壳。**

### ④(可选)family 默认 / 子类型
若 RRAM 有双极/单极等子类型默认,放 `ProtocolSpec.family_defaults`;
读网格、停门度量列等差异项在适配层处理,壳不感知。

### ⑤ 配测试(每个新 runner / 新 schema 各一条)
- runner:dry-run 字节级 golden(参考 `tests/test_engine_run.py`)。
- 绘图器:喂一份小 CSV 不抛(参考 `tests/test_extensibility_seam.py`)。

## 2. 边界(诚实说明:什么时候只能写代码、什么时候纯配置)

- **纯配置(零代码)**:只用已有"积木 step"重排 + 参数组合(见配方 DSL,M5 落地后),
  适合"已有读写动作的新组合"。
- **必须写代码**:新物理原语(C-V、PUND、新波形)、循环/扫描族(笛卡尔积、检查点分块)、
  新仪器/新通道类型、输出新 `csv_schema` 的画法。加一种**全新存储器**通常落在这一类
  (要写 runner + 绘图器),但**壳和功能键、界面布局完全复用,不重写**。

## 3. 自检清单

- [ ] 协议树是否自动多出新 family 分组?(没有 → 检查 `ProtocolSpec.family` 是否注册进 REGISTRY)
- [ ] 选中新协议,参数表单是否按 `params` 自动出控件?(没有 → 检查 ParamSpec 字段)
- [ ] dry-run 跑完是否出图?(没有 → 检查 `@register_plot(csv_schema)` 是否注册、`adapters/__init__` 是否 import)
- [ ] `gui/` 壳层(顶层 .py,排除 `adapters/`)是否一行没改?(改了 → 说明缝没走对,把存储器专有逻辑挪进 `adapters/` 或 `src/fefetlab/`)
- [ ] `tests/test_extensibility_seam.py` 仍绿?(它守"壳层无 FeFET 硬编码 + 新 family 自动出现")
