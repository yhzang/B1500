红框那个口里实际可用的是 16 路 TTL 数字 I/O；另外两个 BNC(Ext Trig In/Out)是单独的触发口。官方文档完整覆盖、完全可编程，跟你项目现有的 GPIB/VISA 链路是同一条，只是多发几条文本命令。

02概念澄清（动手前必读）

⚠ 这 16 根线不能"配置/驱动" SMU/WGFMU 的测量功能 施压、量程、波形那些永远是软件经仪器总线设的（就是你现在代码在干的事）。Digital I/O 干的是外设控制 + 同步，不是替代总线去设测量。

它能干的三件事：

用途说明命令
① 输出驱动板子输出 16 位 TTL 去翻你自制板上的继电器，把 SMU/WGFMU/SPGU 切到不同 DUT 脚ERC
② 读外部状态读 16 位 TTL 做安全联锁：温控 ready / 探针 contact / interlock 异常就停ERM+ERS?
③ 硬件触发每根可配成 trigger 口，零软件延时触发/同步主机测量TGP+OSX/PAX
✓ 所以你的想法成立 "SMU+WGFMU 全插自制板 → B1500 用 16 根 DIO 控板 → 板子路由到器件"——这正是这个口的设计本意。官方 16440A 选择器就是经 16445A 适配器挂这个口、B1500 翻 DIO 位切 SMU↔SPGU 的（你 PPT 里那条"官方控制线"）。你做的是它的通用模式 DIY 版。

03接线准备 · 买什么 / 怎么插

注意：那是一个 25 针连接器（插一个插头），不是 16 个独立口。一个插头里就带了全部 16 根 DIO + GND；16 根线在你这一侧散出来。

买什么（三选一，推荐第一个）

方案淘宝搜索词说明
A. DB25 公头转接线端子板 推荐DB25 公头 转接线端子一头 DB25 插 B1500，另一头 25 个带编号螺丝端子。约 ¥20~40，免焊。
B. DB25 公头焊杯+外壳DB25 公头 焊接 带外壳会焊线最便宜，焊出要用的几针即可。
C. DB25 公头转杜邦线DB25 公头 转杜邦线拿到用万用表通断挡确认每根线对应第几针。
⚠ 买的时候别买错两点 ① 要 公头（有针 pin）——B1500 那侧是母座（孔）。
② 要 标准 DB25（两排 13+12），不是 HD 高密度 25 针（三排）。跟老电脑打印机/并口(LPT)同款外形，认准"两排"。

怎么插

  • 端子板插进 B1500 那个口，拧紧两颗固定螺丝。
  • 只用到：DIO 在端子 3~10、15~22；GND 在端子 13 或 25。
  • 万用表黑笔→端子 13(GND)，红笔→端子 15(DIO1)，准备量电平。

05电气特性（每根 DIO，手册 Figure 2-45）

逻辑电平TTL；高 ≈ 2.4 V，低 ≈ 0.8 V（实际拉低可到 ~0.3 V）
内部结构Vcc = 5 V，上拉 R1 = 1 kΩ，串联 R2 = 100 Ω，下管 Q1（Vce(sat)=0.3 V）→ 准开集（open-collector 带 1kΩ 上拉）
驱动能力弱。能把线拉低、可线或；高电平靠 1kΩ 上拉。不能直推继电器线圈，要加缓冲。
板侧逻辑用 5V TTL；做输入时给标准 TTL 电平。

06控制命令速查

两组：数字口三件套（ERMOD / ERM / ERC / ERS?，控你的继电器板）+ 触发四件套（TGP / OSX·OS / PAX·WSX，做硬件同步）。下方一句话速览，点每条命令展开看完整参数、例子、手册出处。

⚓ 位/权重锚点（命令里的数都从它来，先记这个） DIO n 的十进制权重 = 2ⁿ⁻¹：DIO1=1，DIO2=2，DIO3=4，DIO4=8 … DIO8=128 … DIO16=32768。
要对 DIO a~b 一起操作，把它们的权重相加就是命令里的数（例 DIO1~8 = 1+2+…+128 = 255）。
手册引脚表写作「DIO n (bit n)」——那个 bit n 是从 1 数的第 n 位，最低位就是 DIO1；命令示例里偶尔又写「port 0~15」（从 0 数）。同一根线、两种数法，认权重不会错。
命令一句话
ERMOD mode[,opt]选这 16 根线整体的控制模式（通用 / 各官方选择器）
ERM iport设每根线是输入还是输出
ERC 2,value设输出电平（高/低，⚠ 反逻辑）
ERS?读回 16 根线状态
TGP port,term,pol[,type]把某口登记成触发口
OSX port[,level] / OS从某口主动发触发
PAX port[,wait] / WSX停下等某口的触发再继续

展开详查

ERMOD mode[,option] — 选 16 根线整体当什么用

选数字 I/O 的控制模式。控你自制板 = 通用模式 ERMOD 0（之后才能用 ERM/ERC/ERS?/TGP）。

mode0=通用（初始，自己用 ERM/ERS?/ERC/TGP 控）｜1=16440A 选择器（经 16445A 适配器，改用 ERSSP）｜2=N1258A/N1259A｜4=N1265A｜8=N1266A｜16=N1268A｜32=N1272A
option0=禁用该模式｜1=启用（默认）
ERMOD 0      # 通用模式，控自制板就这条
ERMOD 1      # 进 16440A 选择器模式（之后用 ERSSP）

出处：编程手册 4-101~4-102。

ERM iport — 设每根线方向（输入/输出）

前提 ERMOD 0。改方向后该线被设成 TTL 高；*RST / device clear 后 16 根全部为输出、TTL 高。

iport0~65535 的位图。位=0→输出口，位=1→输入口。某 DIO 的位 = 它的权重 2ⁿ⁻¹。
ERM 0        # 16 根全输出（控板子常态）
ERM 255      # 255=1+2+…+128 → DIO1~8 设输入，其余输出
ERM 32768    # 仅 DIO16 设输入

出处：编程手册 4-100~4-101，原文例 ERM 255 = 二进制 0000000011111111。

ERC 2,value — 设输出电平（⚠ 反逻辑）

前提 ERMOD 0 且对应位是输出。第一个参数固定 2，第二个才是电平位图。

value0~65535 位图。位=0→TTL 高（≈2.4V），位=1→TTL 低（≈0.8V）——和直觉相反。某 DIO 的位 = 权重 2ⁿ⁻¹。
ERC 2,0       # 16 根全高
ERC 2,1       # 仅 DIO1 拉低（其余高）
ERC 2,3       # DIO1+DIO2 拉低（1+2）
ERC 2,65535   # 全部拉低

出处：编程手册命令参考 ERC。翻继电器板最常用的一条。

ERS? — 读回 16 根线状态

查询命令，返回当前 16 路状态位图（十进制）。读输入口（外部状态 / 联锁信号）用。

ERS?          # 返回如 255 → 解析每一位看高/低

出处：编程手册命令参考 ERS?。

TGP port,terminal,polarity[,type] — 把某口登记成触发口

平时一根 DIO 是普通电平（ERC 控）；发 TGP 后它变成触发线，能收/发触发脉冲做硬件同步（零软件延时）。

port1~16=数字口 DIO1~16｜-1=Ext Trig In（后面板 BNC）｜-2=Ext Trig Out（BNC）
terminal1=触发输入（收；port=-2 不可）｜2=触发输出（发；port=-1 不可）
polarity1=正逻辑｜2=负逻辑（初始）
type默认 0。0=基础收/发（配 PA/WS/PAX/WSX/OS/OSX）｜1/2/3=与测量挂钩的自动触发（开始测量 / 测量完成 / 每扫描步同步等，需先发 TM3）
TGP 1,1,1      # DIO1 当触发输入，正逻辑，基础型(type0)
TGP 2,2,2      # DIO2 当触发输出，负逻辑
TGP -1,1,2     # Ext Trig In(BNC) 当触发输入，负逻辑

出处：编程手册 4-212~4-213（type 含义见 Table 4-31）。

OSX port[,level] / OS — 从某口主动发触发

从指定触发口发一个触发出去（前提：该口已用 TGP 设成触发输出）。OS 是简版，只能从 Ext Trig Out 那个 BNC 发边沿触发。

port-2=Ext Trig Out（BNC）｜1~16=数字口（须与 TGP 设的口一致）
level0=逻辑低｜1=逻辑高｜2=边沿触发（默认）。门控触发：开始 OSX port,1、结束 OSX port,0
OS            # 从 Ext Trig Out(BNC) 发一个边沿触发
OSX 1         # 从 DIO1 发边沿触发（DIO1 须先 TGP 设成输出）

出处：编程手册 4-162。配对命令是 PAX/WSX（一个发、一个等）。

PAX port[,wait] / WSX — 停下等某口的触发再继续

OSX 的反面：让 B1500 暂停命令 / 程序执行，直到等到指定口的触发（或超时）再继续。WSX 是类似的「进入等待态直到收到触发」。需配 TM3。

port-1=Ext Trig In（BNC）｜1~16=数字口（须与 TGP 一致）
wait-99.9999~99.9999 秒。不写或负值 = 一直等到触发来
PAX 1         # 停下，等 DIO1 来触发（一直等）
PAX -1,10     # 等 Ext Trig In，最多等 10 秒

出处：编程手册 4-164。

07测试步骤（先验 16 根线，不接器件、零风险）

1
接好：DB25 公头端子板插进 Digital I/O 口拧紧。万用表黑笔→端子13(GND)，红笔→端子15(DIO1)。本机资源串 GPIB1::17::INSTR（或 autodetect_visa_addr("B1500")）。
2
进通用模式：发 ERMOD 0。
3
全高→全低：ERC 2,0 量 DIO1 应 ≈2.4V↑；ERC 2,65535 应 ≈0.8V↓。电压跟着命令跳 = 控得动。
4
实测确认引脚映射：发 ERC 2,1（只拉低 DIO1），哪个端子电压掉到 ~0.8V，那个就是 DIO1。逐位 ERC 2,2/4/8… 核对 DIO2/3/4…，不靠猜。
5
读回：ERS? 看状态位图。收尾 ERC 2,0 全高复位。
内置自检 DIAG? 4 = Digital I/O 自检（返回 0=过）。跑前务必拔掉这个口所有外接线。　DIAG? 1 = Trig In/Out 自检（需 BNC 把 Ext Trig In↔Out 短接）。

08测试代码 scripts/digital_io_smoke.py

套现有 pyvisa 链路，纯翻 TTL、逐根验、读回，不接器件：

# digital_io_smoke.py — B1500 Digital I/O 16 路 TTL 自检（不接器件，纯量电平）
import pyvisa, time

ADDR = "GPIB1::17::INSTR"   # 或用 autodetect_visa_addr("B1500")

# DIO n -> 端子(针)号，来自手册 Table 2-6
PIN = {1:15,2:3,3:16,4:4,5:17,6:5,7:18,8:6,9:19,10:7,11:20,12:8,13:21,14:9,15:22,16:10}

def bit(n):           # DIO n 的位权（DIO1=bit0）
    return 1 << (n - 1)

rm = pyvisa.ResourceManager()
inst = rm.open_resource(ADDR)
inst.timeout = 10000
try:
    inst.clear()                       # 轻 preflight
    while inst.query("ERRX?").strip().split(",")[0] not in ("0","+0"):
        pass                           # 排空错误队列
    print("IDN:", inst.query("*IDN?").strip())

    inst.write("ERMOD 0")             # 通用控制模式
    inst.write("ERC 2,0")              # 16 路全部输出高
    print("全高，量 DIO1(针15) vs GND(针13) 应≈2.4V+"); time.sleep(1)

    # 逐根拉低，确认端子映射
    for n in range(1, 17):
        inst.write(f"ERC 2,{bit(n)}")    # 只把 DIO n 拉低，其余高
        print(f"DIO{n:<2} 拉低 -> 量针{PIN[n]} 应≈0.8V   ERS?={inst.query('ERS?').strip()}")
        time.sleep(0.8)

    inst.write("ERC 2,65535"); print("全低"); time.sleep(1)
    inst.write("ERC 2,0")              # 收尾全高复位
    print("完成。")
finally:
    inst.close(); rm.close()
✓ 安全 这段只切换 16 根 TTL 电平、读状态，不接任何器件、不动 SMU/WGFMU 输出，零风险。验完即证明这 16 根线你能随意控了。

09板子架构 · 工程红线

目标板 = 选择器/开关矩阵板：SMU + WGFMU 测量线全插板上，B1500 用 16 根 DIO 翻继电器，把对应源路由到器件。架构成立、是正路。真正难点只在信号完整性。

两套线分开（核心）

控制路径16 根 DIO（弱 5V TTL）→ 只决定"哪个继电器闭合"
信号路径SMU triax(力/感+guard)、WGFMU 同轴脉冲 → 不经 DIO，单独插板、由继电器切到 DUT

四条工程红线

① SMU 侧保 guard + Kelvin 继电器带来漏电/电缆电容/丢 guard，毁低电流精度(pA~nA)。用低漏/干簧(reed)/带保护继电器，布局保 guard。FeFET 到 µA 级可放松，但要心里有数。
② WGFMU 快脉冲最怕继电器 stub µs/ns 快沿被继电器寄生电容/电感糊掉。现拓扑 baseline ~1.14µA 本就是 3m HRSU ~150pF 寄生主导，再串继电器更糟。Keysight 的解法是 RSU（探针端切，现有 3 个 RSU 就是它）。→ 切 DC/SMU 路由或"SMU↔WGFMU 模式"OK；把 WGFMU 快测路径塞进通用继电器板有风险，用射频/干簧继电器+极短 stub 或照 RSU 思路，先单独验快沿。
③ 继电器驱动 DIO 弱上拉不能直推线圈 → 板上 ULN2003/MOSFET/光耦 + 独立线圈电源。16 根线最多直驱 16 个继电器(1bit/个)；要更多加移位寄存器/IO 扩展。
④ 切源 break-before-make 先断后通，别把两个源短一起；软件里按位序发 ERC 控制时序。

官方参照：16440A 是什么

Keysight 16440A = 官方 SMU/脉冲发生器选择器（一个继电器盒），两路输入(SMU + SPGU 脉冲)，由 B1500 经 16445A 选择器适配器插这个口控制(ERMOD 1+ERSSP)，把器件脚在 SMU↔SPGU 间切换。它就是你要自制那块板的官方商品版。
